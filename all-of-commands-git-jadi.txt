git init
	 Initializes a new Git repository in the current folder.

git commit
	 Records the staged changes as a new commit in the repository.

git add
	 Adds files or changes to the staging area, preparing them for the next commit.

git add "(similar-name)*"
	 Adds all files whose names match the specified pattern. Wildcards (*) can also be used with other Git commands.

git add -A
	 Stages all new, modified, and deleted files in the current project.

git status
	 Displays the status of your working directory, including modified, new, staged, and committed files.

git log
	 Shows the commit history, including commit information and commit messages.

git diff HEAD
	 Compares your current working directory with the latest commit (HEAD) and displays all differences, whether they are staged or unstaged. You can also compare against any other commit by specifying its commit hash.

git diff --staged
	 Shows the differences between the staged files and their last committed versions.

git reset (file name)
	 Removes the specified file from the staging area without discarding its changes.

git checkout -- (file name)
	 Restores the specified file to its state from the last commit, discarding any unstaged changes.

git branch
	 Lists all branches in the repository.

Branch
	 A branch is an independent line of development created from a commit. It allows you to add features or make fixes without affecting the main branch until you merge the changes.

Example:
	 Suppose you have several commits. You can create a branch from Commit 1, add new changes there, and later merge them into another branch that already contains Commit 2.

git branch (name)
	 Creates a new branch.

git checkout (branch name or tag)
	 Switches to the specified branch or tag. You can make changes on a branch, but a checked-out tag is in a detached state and should not be modified directly.

git merge (branch name)
	 Merges the specified branch into the current branch.

"(similar-name)*"
	 A wildcard pattern that matches multiple files with similar names. It can be used with commands such as git add, git reset, and others.

git rm (file name)
	 Removes the specified file from both Git tracking and the current branch.

git branch -d (branch name)
	 Deletes the specified branch.

cat (file name)
	 Displays the contents of a file in the terminal.

cd ..
	 Moves to the parent directory.

git push origin (branch name or tag)
	 Uploads commits from your local repository to the specified remote repository.

git pull origin (branch name or tag)
	 Downloads and merges updates from the specified remote branch or tag into your local repository.

git push origin --tags
	 Pushes all local tags to the remote repository.

git pull origin --tags
	 Downloads all tags from the remote repository.

git remote
	 Manages remote repositories. Remotes allow you to synchronize your repository with services such as GitHub, GitLab, or other Git hosts.

git remote add (remote name) (repository URL)
	 Adds a new remote repository with the specified name and URL.

git push -u (remote name) (branch name)
	 Pushes the specified branch to the remote repository and sets it as the upstream branch.

git remote -v
	 Displays all configured remote repositories along with their fetch and push URLs.

git pull
	 Fetches and merges changes from the configured upstream remote branch.

git push
	 Pushes commits to the configured upstream remote branch.

git show
	 Displays detailed information about a specific commit or tag. You can provide a commit hash or tag name to view its contents.

git tag
	 Lists all existing tags.

Tag
	 A tag is a reference to a specific commit, typically used to mark release versions. Unlike branches, tags are fixed and are not intended for further development.

git tag -a (tag name) -m "(message)"
	 Creates an annotated tag with the specified message.

git tag -a (tag name) (commit hash) -m "(message)"
	 Creates an annotated tag for a previously created commit.

git tag -l "(pattern)*"
	 Lists tags whose names match the specified pattern.

GPG
	 GNU Privacy Guard (GPG) is software used for digital signatures and encryption. It verifies your identity and ensures the authenticity of commits and tags.

gpg --list-keys
	 Lists all GPG public keys available on your system.

gpg --gen-key
	 Generates a new GPG key pair for digital signing.

git config --global user.signingkey (key ID)
	 Configures Git to use the specified GPG key for signing commits and tags.

gpg --list-keys --keyid-format LONG
	 Displays your GPG keys using the long key ID format. The long key ID is typically used when configuring Git.

git tag -s (tag name) -m "(message)"
	 Creates a signed tag using your configured GPG key.

git tag -v (tag name)
	 Verifies a signed tag and displays who signed it and whether the signature is valid.

git commit -S -m "(message)"
	 Creates a GPG-signed commit.

GPG note
	 If commit or tag signing is required in a project, every contributor should sign their commits and tags.

git help (command)
	 Displays the complete documentation and usage information for the specified Git command.

git blame
	 Shows who last modified each line of a file.

git blame (file name)
	 Displays the last author and commit responsible for every line in the specified file.

git blame (file name) -L(start,end)
	 Shows blame information only for the specified line range.
Example:
git blame file.txt -L8,10

git bisect
	 Starts the Git bisect process, which uses binary search to locate the commit that introduced a bug.

git bisect bad
	 Marks the current commit as containing the bug.

git bisect good (commit hash)
	 Marks the specified commit as known to be free of the bug. Git then continues the binary search to identify the first bad commit.
