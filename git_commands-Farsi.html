دستورات گیت

General Commands (دستورات عمومی)

  • git config --global user.name "USERNAME" : پیکربندی نام کاربری در کل سطوح
  • git config --global user.email "USEREmail" : پیکربندی ایمیل کاربری
  • git config --global color.ui true : نمایش خروجی رنگی
  • git init: شروع فعالسازی گیت در دایرکتوری فعلی
  • git status: نمایش وضعیت الان گیت
  • git add: فایل های ایجاد شده یا تغییر کرده را به استیج اضافه میکند
  • git add . & git add -A: این دو دستور تمامی فایل های ایجاد شده یا تغییر کرده را به استیج اضافه میکند
  • git commit: ثبت کردن تغییرات و حذف کردن فایل ها از استیج
  • git commit -m "Title" -m "message" : ثبت کردن کامیت با عنوان و پیام تغییرات
  • git log: نمایش دادن وقایع اتفاق افتاده با توضیحات
  • git log --one-line --graph --all --decorate: نمایش دادن وقایع به صورت درختی با همه شاخه ها و غیره
  • git diff <...>: مقایسه دو نسخه از یک فایل
      • برای مثال :
    • git diff HEAD: مقایسه آخرین نسخه تایید شده با نسخه فعلی
    • git diff --staged: مقایسه آخرین نسخه استیج شده با نسخه فعلی
  • git reset filename: خارج سازی فایل از استیج
  • git reset commitHash: بازگشت به کامیت مشخص شده با هَش
  • git reset commitHash --hard: پروژه به طور کامل به کامیت مشخص شده بازگشته و تغییرات حذف خواهند شد
  • git checkout -- filename: فایل را به آخرین کامیت برمیگرداند
  • git rm filename: حذف فایل از سیستم و گیت

Branches (شاخه ها)

  • git branch: (برنچ ها) نمایش تمامی شاخه ها
  • git branch -d branchName: پاک کردن شاخه یا شاخه ها
  • git checkout branchName: وارد شدن به شاخه مشخص شده
  • git checkout -b branchName: ساختن و وارد شدن به شاخه مشخص شده
  • git merge branchName: ادغام کردن شاخه مشخص شده با شاخه مستر

Remote & Clone (ریموت و کلون)

  • git clone gitAddress: کلون (کپی کردن) کردن کامل یک ریپوزیتوری یا آدرس
  • git remote add alias gitAddress: (alias یک اسم مستعار است) اضافه کردن یک ریموت
  • git remote -v: نمایش کامل اطلاعات ریموت ها
    • توجه: ما میتوانیم به هر تعداد ریموت داشته باشیم (یکی برای گیت لب ، یکی برای گیت هاب و یکی برای شرکت)
    origin: این ریموت ، شاخه اصلی پروژه ای است که کپی کردیم
  • git push origin master: وارد کردن تغییرات از شاخه مستر به شاخه اصلی
    • توجه: از تاریخ ده مهر 1399 هر مخزن جدید ، از main بجای شاخه مستر استفاده میکند
    پس اگر میخواهید شاخه جدیدی اضافه کنید از دستور زیر استفاده نمایید :
    git push origin main
    توجه: git push -u origin master: سوییچر u- باعث ذخیره سازی اطلاعات میشود ، پس در ادامه دیگر نیازی به نوشتن master main وجود ندارد
  • git pull origin master: وارد کردن تغییرات از شاخه اصلی به مستر (برعکس push)
    • توجه : git remote add فقط یک ورودی در پیکربندی git شما ایجاد می کند که نام یک URL خاص را مشخص می کند. برای استفاده از آن باید یک مخزن git موجود داشته باشید.
    git clone : یک مخزن گیت جدید با کپی کردن یک مخزن موجود واقع در لینک که شما مشخص کرده اید ایجاد می کند.

Tag (برچسب)

  • git tag: نمایش برچسب های موجود
  • git tag -a <...> -m '' : ایجاد تگ با اسم و یادداشت مانند زیر :
    git tag -a v2.0 -m 'release Version 2.0'
  • git tag -a <...> commitHash -m '' : ایجاد تگ برای یک کامیت مشخص
  • git tag -v <...> verifytag -v : نمایش برچسب های تایید شده
  • git show tagName: نمایش کامل اطلاعات برچسب
  • git push origin tagName: برچسب ها به صورت خودکار ست نمیشوند و باید دستی این کار را کنید
  • git push origin --tags: وارد کردن همه برچسب ها به شاخه اصلی
  • git blame : بررسی اینکه کل کد یا بخشی ازش یا حتی خطی ازش توسط کیا نوشته و ویرایش شده
  • pgp (gpg نسخه متن باز آن است) : ساین کردن تگ ها یا کامیت ها

Bisect

  • git bisect : اشکال زدایی با استفاده از حرکت بین کامیت ها
  • git bisect start : شروع کار دیباگ
  • git bisect bad : انتخاب کامیتی که خطا دارد (یا در حال حاضر خطا دارید)
  • git bisect good : انتخاب کامیتی که در آن خطا ندارید
توجه: هر بار که از bisect استفاده می کنید باید کد خود را آزمایش کنید و بعد از git bisect از good یا bad استفاده کنید
